# デイトレードシミュレーター仕様書

最終更新日: 2026-07-05

この文書は、現在のアプリを後からゼロから再作成するための仕様書です。以降で追加要望や仕様変更があった場合は、このファイルの該当セクションと末尾の変更履歴を必ず更新します。

## 1. 目的

過去の株価データを使い、デイトレードの練習とルールベース売買アルゴリズムの検証を行うローカルアプリを作る。

本アプリは学習・検証用であり、実売買APIには接続しない。表示価格、約定、損益、レポートはすべてシミュレーションであり、実取引の結果を保証しない。

## 2. 想定利用者

- デイトレード練習をしたい個人ユーザー
- 1分足から60分足までの短期足と、日足・週足・月足の長期足を見ながら裁量判断を練習したいユーザー
- VWAP、移動平均線、ボリンジャーバンドなどを使ったルールベース戦略を検証したいユーザー

## 3. 技術スタック

- 言語: Python 3.11
- UI: Streamlit
- チャート: Plotly
- データ処理: pandas、numpy
- 実データ取得: yfinance
- バリデーション: pydantic
- レポート: matplotlib、Plotly、HTML/CSV
- テスト: pytest
- 静的チェック: ruff
- 環境管理: conda、`environment.yml`、`requirements.txt`

## 6. データ仕様

### 6.1 データ種別

サイドバーで以下を選択できる。

- 実データ（Yahooファイナンス）
- サンプル

デフォルトは実データ。

### 6.2 Yahooファイナンス実データ

- `yfinance` を使って取得する。
- デフォルト銘柄はキオクシアホールディングス `285A` とする。
- デフォルト日付は、`285A` のYahoo日足キャッシュがある場合はその最新日、ない場合は今日以前の直近平日とする。
- 東証銘柄は、`285A` のように入力された場合、内部で `285A.T` に変換する。
- 1分足は対象日1日分を取得する。
- Yahooの1分足が9:00より後から始まる場合、9:30までの欠損に限り、9:00から実データ先頭直前まで出来高0の補完足を追加する。補完足の価格は対象日の日足始値を優先し、日足始値がない場合は取得できた最初の1分足始値を使う。
- Yahooから取得する日足キャッシュには対象日までの過去約180日分を保存できる。
- デイトレード画面、重要価格ライン設定画面、日足・週足・月足チャートでは、対象日の前日までの日足だけを表示する。対象日当日以降の日足は、当日の練習中に知り得ない情報として表示しない。
- 取得済みデータは `data/raw/yahoo/minute/` と `data/raw/yahoo/daily/` にキャッシュする。
- `実データを再取得` ボタンでキャッシュを無視して再取得する。
- Yahooファイナンスの1分足は取得可能期間が短いため、対象日・銘柄によって取得できない場合がある。
- 実データ取得に失敗した場合、サンプルへ黙ってフォールバックせず、日本語エラーを表示して停止する。

### 6.3 サンプルデータ

- `data/raw/minute/` と `data/raw/daily/` にサンプルCSVがなければ自動生成する。
- キオクシアホールディングス `285A` は、2026-06-24付近で約90,000円台になるように生成する。

### 6.4 CSVスキーマ

1分足:

```csv
timestamp,open,high,low,close,volume
2026-06-24 09:00:00,90000,91080,89280,90540,120000
```

日足:

```csv
date,open,high,low,close,volume
2026-06-24,90000,91800,88200,90750,18000000
```

価格単位は円。UI上では `93,730円` のようにカンマ区切りで表示し、`k` 表記は使わない。

## 7. ReplayEngine仕様

- 売買練習中に未来の1分足を見せない。
- `current_state()` は現在時点までの分足だけを返す。
- 対象日当日以降の日足は観測結果に含めない。
- 前日までの日足がない場合でも、当日の日足を代替表示しない。
- 前場と後場の間はチャートをリセットせず、x軸上では連続した足として表示する。
- `1分戻る` 操作では、直前の時刻、建玉、約定履歴、確定損益、完了状態をスナップショットから復元する。
- データ末尾に到達したら `done=True` になる。
- `force_close_on_end=True` の場合、最後に未決済建玉を自動決済する。

## 8. 売買・口座仕様

### 8.1 アクション

- 買い
- 空売り
- 決済
- 1分戻る
- 1分進める
- 再生
- 一時停止
- リセット
- 一括検証

### 8.2 約定

- 現在足の終値で成行仮想約定する。
- 買い建て中に買いを押すと、買い増しする。
- 空売り建て中に空売りを押すと、売り増しする。
- 同方向の追加建玉は平均建値を再計算して1つの集約ポジションとして扱う。
- 買い建て中に空売りを押した場合、現在の買い建玉を全決済する。
- 空売り建て中に買いを押した場合、現在の空売り建玉を全決済する。
- 決済ボタンは現在の集約ポジションを全決済する。
- 部分決済、ロット別管理、手数料、税金、スリッページ、信用取引規制は現時点では実装しない。

### 8.3 入金額と余力

- 初期入金額はサイドバーで入力する。
- デフォルト入金額は10,000,000円。
- 注文株数はサイドバーで入力する。
- デフォルト注文株数は100株。
- 必要額は `約定価格 * 注文株数` とする。
- 買付余力は `初期入金額 + 確定損益 - 建玉金額` とする。
- 必要額が買付余力を超える場合、日本語エラーを表示して注文を拒否する。
- 口座表示は必要十分な情報に絞り、売買画面では `評価額`、`買付余力`、`建玉金額` をメトリクス表示する。
- `確定損益` と `含み損益` は画面上部の主要メトリクスに表示し、口座欄では重複表示しない。

## 10. 重要価格ライン設定仕様

画面上の正式名称は `重要価格ライン` とする。`ネックライン` はチャートパターン上の特定ラインを指す意味が強いため、サポートライン、レジスタンスライン、直近高値、出来高の多い価格帯などをまとめる総称としては使わない。

### 10.1 起動時フロー

1. アプリ起動後、まず参照用の日足チャートを必ず表示する。
   - ただしサイドバーの `重要価格ラインを使う` がOFFの場合は、この設定画面を表示せずデイトレード画面へ直接進む。
2. ユーザーは日足チャートを見ながら、サポートライン、レジスタンスライン、直近高値、直近安値、移動平均線、ボリンジャーバンド、出来高の多い価格帯などの基準ラインを選ぶ。
3. ラベルと色を選ぶ。
4. 日足チャート上にカーソルを乗せると、カーソル位置の価格に水平の点線ガイドを表示する。
5. 日足チャート上の任意の価格を単純クリックすると、その価格に水平の重要価格ラインを追加する。
6. クリック時の価格は、チャート上で選択された価格として即時に反映する。
7. 任意本数のラインを追加できる。
8. `最後のラインを削除`、`全ラインを削除` で編集できる。
9. `完了してデイトレードへ` を押すとデイトレード画面へ進む。

### 10.2 ラベル

選択肢:

- サポートライン
- レジスタンスライン
- 直近高値
- 直近安値
- 移動平均線
- ボリンジャーバンド
- その他

ラベル候補を選ぶと、その名前が自由入力欄の初期値になる。ユーザーは候補に関係なく、任意のラベル名へ編集できる。

### 10.3 表示

- 設定した重要価格ラインは、分足チャート、日足、週足、月足に常時表示する。
- `重要価格ラインを使う` がOFFの場合、設定済みのラインがあってもチャートには表示しない。
- `重要価格ラインを使う` がOFFの場合、銘柄・日付・データ種別を変更しても重要価格ライン設定画面へ戻らない。
- ラインにはラベルと価格を表示する。
- 価格は円表記にする。
- 重要価格ライン設定画面では、Plotlyの日足チャート自体を価格選択UIとして使う。
- ホバー中は価格位置に水平点線ガイドを表示し、クリックで選択価格を取得する。
- 価格取得は範囲選択ではなく単純クリックを主操作にする。Plotlyはポイント選択モードだけを有効にし、ドラッグモードはpanにして、クリックが価格選択として発火しやすい状態にする。
- クリックイベントや選択イベントにローソク足など複数点が混ざる場合でも、価格選択用の透明クリックレイヤーの点を優先して価格を取得する。
- チャート右側に、日足の高値から安値へ出来高を配分した価格帯別出来高を横棒グラフで表示する。
- 日足チャート上部に、表示範囲を直接動かせるインタラクティブな日足表示範囲バーを置く。
- 日足表示範囲バーは、Streamlitの範囲スライダーとして実装する。日付を直接入力するフォームではなく、左右のつまみをドラッグして開始位置と終了位置を変更する。
- スライダーの内部値は、対象日より前に取得できた日足データの0始まり営業日インデックスとする。画面には `選択範囲: YYYY-MM-DD 〜 YYYY-MM-DD` のキャプションを表示し、現在の選択日付を確認できるようにする。
- 日足表示範囲バーの画面上の目盛りとつまみラベルは、日数インデックスではなく `YYYY-MM-DD` 形式の日付で表示する。
- 日足表示範囲バーは営業日だけを選択肢にし、土日や休場日などデータが存在しない日付を選ばない。
- 日足表示範囲バーで範囲を選択した場合、日足チャート、クリック対象価格、価格帯別出来高は選択範囲に合わせて再計算・再表示する。
- 日足表示範囲バーはチャート本体と横幅が完全一致していなくてもよい。操作しやすさを優先する。
- 範囲選択後は、日足チャート本体、価格選択用の透明クリックレイヤー、右側の価格帯別出来高を同じ選択範囲で再計算・再描画する。
- 価格選択用の透明クリックレイヤーは、表示中の日足全体を対象にし、表示範囲の左側や120営業日より前の範囲でもクリックで価格を取得できるようにする。
- `直近60営業日に戻す` ボタンを用意し、選択範囲を取得済み日足データの末尾60営業日へ戻せるようにする。
- チャート内には日足表示範囲を示す青い補助バーや青い帯を表示しない。
- 価格選択用の透明クリックレイヤーは視覚的に見えない完全透明表示にし、日足チャートの背景は白地、gridは薄いグレーにする。青と白の縞模様に見える表示は避ける。
- 重要価格ライン設定画面の日足チャートと通常の日足チャートでは、価格目盛りが見切れないよう左余白を十分に確保し、y軸目盛りは自動余白を有効にする。

## 11. チャート仕様

### 11.1 分足チャート

- 1分足データを元に、表示時に3分足、5分足、10分足、30分足、60分足へリサンプルする。
- リサンプル時のOHLCV:
  - open: 最初の始値
  - high: 最大高値
  - low: 最小安値
  - close: 最後の終値
  - volume: 合計出来高
- 分足ごとに移動平均線とボリンジャーバンドを再計算する。
- 約定判定は引き続き1分足単位で行う。
- 表示範囲は現在までの過去10分、30分、60分、または全表示。
- `過去10分`、`過去30分`、`過去60分` は、開始直後からx軸レンジを固定幅にする。
- 時間が進んでもローソク足の横幅が狭くならないよう、選択した表示範囲の幅を常に一定に保つ。
- 現在時刻の足を右端ぴったりに置かず、右側に少し余白を作って表示する。
- `全表示` は例外として、再生済みの全範囲を表示するため時間経過に応じて横幅が増える。
- 前場と後場の間はチャートをリセットせず、x軸上ではつなげて表示する。
- 自動再生中に前場最後の足から後場最初の足へ進む場合は、5秒待機して昼休みをまたいだことを表現する。
- 開始直後の分足チャートでは、前日の日足ローソクを左側に部分表示し、前日から当日寄付きへの文脈を確認できるようにする。
- 前日の日足ローソクは初期表示の文脈としてのみ価格レンジ計算に含める。再生が始まり当日分足が2本以上見えている場合は、前日の日足ローソクを価格レンジ計算に含めず、当日分足が見やすい価格帯を優先する。
- デイトレード画面の出来高バーは、日本株チャートの慣例に合わせて、対応するローソク足が陽線なら赤、陰線なら緑、同値ならグレーで表示する。
- 分足ローソク足本体も、陽線は赤、陰線は緑にする。
- 出来高バーは、各足のローソク足と同じ方向判定から作った色配列を `marker.color` と `marker.line.color` に明示して描く。陽線は赤、陰線は緑、同値はグレーにする。
- 出来高は1本のBarトレースとして描画し、足ごとの色配列で塗り分ける。色別に複数トレースへ分けたり、Plotlyのデフォルト色に任せたりしない。
- 分足チャートでは、時間、価格、出来高のtickラベルが見切れないよう、左・右・下の余白を十分に確保し、x軸とy軸の自動余白を有効にする。
- 分足チャートのVWAP、移動平均線、ボリンジャーバンドは、表示点数が少ない序盤でもマーカー付きの `lines+markers` にならないよう、常に `mode="lines"` で描画する。
- Yahoo分足の寄り付き付近で、価格が動いている先頭実足だけ出来高0になっている場合は、次の非ゼロ出来高を表示用フォールバックとして使い、寄り付きの出来高バーが欠落して見えないようにする。
- y軸は現在表示されているローソク足が十分大きく見える価格帯へ動的に調整する。
- 離れすぎた重要価格ラインやボリンジャーバンドは、ローソク足を過度に縮小しないようy軸範囲の計算から除外する。
- 分足のy軸レンジはローソク足の高値・安値を主軸にし、表示中のVWAP、移動平均線、ボリンジャーバンド、重要価格ラインのうち近いものだけを補助的に含める。

### 11.2 長期足チャート

- 日足、週足、月足を表示できる。
- 週足は週末基準で集約する。
- 月足は月末基準で集約する。
- 長期足の移動平均線とボリンジャーバンドは選択的に表示できる。
- 重要価格ラインも常時表示する。

### 11.3 トレードマーカー

- 約定したタイミングをチャート上に残す。
- マーカー位置は該当足の実体中央値、つまり `(始値 + 終値) / 2` とする。
- 買い建て、買い決済、空売り建て、空売り決済で色と形を変える。
- 3分足以上を表示している場合は、約定した1分足を含む集約足にマーカーを表示する。

## 12. 指標仕様

### 12.1 出来高加重平均価格

- 分足に表示する。
- 当日ここまでの累積価格出来高を累積出来高で割って算出する。

### 12.2 移動平均線

分足:

- 5本移動平均
- 25本移動平均

長期足:

- 5本移動平均
- 25本移動平均
- 75本移動平均

### 12.3 ボリンジャーバンド

- 中心線は20本移動平均。
- 標準偏差は母標準偏差 `ddof=0`。
- 表示するバンド:
  - +1σ
  - -1σ
  - +2σ
  - -2σ
  - +3σ
  - -3σ
- 同じσのプラス側とマイナス側は同じ色で表示する。

## 13. バックテスト仕様

CLI:

```bash
python run_backtest.py --symbol 285A --date 2026-06-24 --strategy vwap_ma_breakout
```

## 14. 自動売買戦略仕様

Streamlit UIでは、サイドバーの `自動売買モード` をONにすると、選択中の売買アルゴリズムが手動ボタンと同じ `Action` を自動的に発行する。`再生` 中は1分進むたびに戦略判断を行い、`1分進める` を押した場合もその1分だけ戦略判断を行う。`一括検証` は自動売買モードONの場合、倍速再生を待たずに対象日の最後まで戦略判断を繰り返す。

`一括検証モード` をONにすると、`再生` ボタンでも対象日の最後まで即時実行する。これは売買アルゴリズムの有効性を効率よく確認するためのモードであり、実行後に実行分数、約定数、トレード数、勝率、損益を画面上に表示する。

### 14.1 VWAP + 移動平均ブレイクアウト

- フラット時に、現在値が出来高加重平均価格と5本移動平均を上回り、5本移動平均が25本移動平均を上回る場合に買い。
- 買い建て中に、現在値が出来高加重平均価格または25本移動平均を下回る場合に決済。

### 14.2 ボリンジャー逆張り

- ボリンジャーバンドを使った逆張り戦略として実装する。
- 詳細条件は `src/strategies/bollinger_reversion.py` を正とする。

### 14.3 ボリンジャー3σ逆張り

- 初期の自動売買モード用アルゴリズムとして実装する。
- 詳細条件は `src/strategies/bollinger_next_reversion.py` を正とする。
- 1分足の終値がボリンジャーバンド -3σ を下回った場合、次の1分足で買い建てする。
- 1分足の終値がボリンジャーバンド +3σ を上回った場合、次の1分足で空売り建てする。
- エントリー判定には1本前の1分足を使い、3σ外へ出た足そのものでは入らない。
- 建玉が含み益になった時点で決済する。
- 建玉の含み損が建値比5%以上になった時点でロスカットする。
- ボリンジャーバンドが十分に安定するよう、最低20本の過去足があるまでは新規エントリーしない。

### 14.4 複合ルール

- 複数指標を組み合わせた戦略として実装する。
- 詳細条件は `src/strategies/combined_rule.py` を正とする。

## 16. GitHub管理方針

- コード、仕様書、README、テスト、環境定義をpushする。
- `data/`、`outputs/`、`__pycache__/`、`.pytest_cache/` などの生成物はpushしない。
- 大きな実データやHTMLレポートは必要になった場合だけ別途管理方針を決める。
- 追加要望を実装した場合は、必ずこの仕様書とREADMEの必要箇所を更新する。
- READMEはGitHubで最初に見る入口として短く保ち、細かい機能説明、データ形式、実装仕様はこの仕様書に集約する。
- READMEには実行中画面のデモ動画 `docs/assets/demo_movie.mov` を掲載し、GitHub上でアプリの雰囲気がすぐ分かるようにする。
- READMEのデモ動画はHTMLの `video` タグと直接リンクの両方で掲載し、GitHub上で動画タグが再生できない環境でも開けるようにする。
- `scripts/record_readme_demo.py` と `docs/assets/demo.gif` は、必要に応じてGIF版デモを再生成するための補助資産として残す。
- デモ動画では、日足チャートで価格帯別出来高が多い重要価格ラインを決定する様子と、デイトレード画面で買い、60倍速での値動き確認、買い決済による利益確定までを伝える。
- READMEには代表的な売買戦略候補を簡潔に記載する。
- READMEには実データ検証成績の表を置く。表には銘柄、日付、当日の特徴、前日比、戦略、損益、勝率、メモを記録する。初期状態では空欄でよい。

## 17. 既知の制約

- 実売買APIには接続しない。
- Yahooファイナンスの1分足は取得可能期間が短く、過去日付によっては取得できない。
- 手数料、税金、スリッページ、信用取引の厳密な保証金管理は未実装。
- 集約ポジションのみ扱い、ロット別ポジションや部分決済は未実装。
- 重要価格ラインは水平線のみ。
- 重要価格ライン設定はドラッグ描画ではなく、チャートクリックによる価格指定。
- `現在値にラインを追加` のような補助ボタンは置かず、日足チャートのクリックで重要価格ラインを追加する。
- 自動再生はStreamlitのrerunとsleepによる簡易再生。
- 売買画面の `リセット` は、重要価格ライン設定には戻らず、時刻、建玉、トレード履歴を対象日の最初へ戻す。
- サイドバーの `ライン設定からやり直す` は、`重要価格ラインを使う` がONの場合だけ押せる。押すと重要価格ライン設定画面へ戻る全体リセットとして使う。
- `重要価格ラインを使う` がOFFの場合、日付変更やデータ再取得をしても重要価格ライン設定画面へ自動遷移しない。

11. `src/strategies/` にルールベース戦略を実装する。
12. `run_backtest.py` と `src/analysis/` にCLIバックテストとレポート出力を実装する。
13. `tests/` に単体テストを追加する。
14. `README.md` にセットアップ、使い方、注意事項、Codexを使って作成した旨を記載する。
