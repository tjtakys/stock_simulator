# デイトレードシミュレーター仕様書

最終更新日: 2026-07-04

この文書は、現在のアプリを後からゼロから再作成するための仕様書です。以降で追加要望や仕様変更があった場合は、このファイルの該当セクションと末尾の変更履歴を必ず更新します。

## 1. 目的

過去の株価データを使い、デイトレードの練習とルールベース売買アルゴリズムの検証を行うローカルアプリを作る。

本アプリは学習・検証用であり、実売買APIには接続しない。表示価格、約定、損益、レポートはすべてシミュレーションであり、実取引の結果を保証しない。

## 2. 想定利用者

- デイトレード練習をしたい個人ユーザー
- 1分足から60分足までの短期足と、日足・週足・月足の長期足を見ながら裁量判断を練習したいユーザー
- VWAP、移動平均線、ボリンジャーバンドなどを使ったルールベース戦略を検証したいユーザー

## 3. 技術スタック

- 言語: Python 3.11
- UI: Streamlit
- チャート: Plotly、ネックライン設定用のStreamlitカスタムコンポーネント
- データ処理: pandas、numpy
- 実データ取得: yfinance
- バリデーション: pydantic
- レポート: matplotlib、Plotly、HTML/CSV
- テスト: pytest
- 静的チェック: ruff
- 環境管理: conda、`environment.yml`、`requirements.txt`

## 6. データ仕様

### 6.1 データ種別

サイドバーで以下を選択できる。

- 実データ（Yahooファイナンス）
- サンプル

デフォルトは実データ。

### 6.2 Yahooファイナンス実データ

- `yfinance` を使って取得する。
- 東証銘柄は、`285A` のように入力された場合、内部で `285A.T` に変換する。
- 1分足は対象日1日分を取得する。
- 日足は対象日までの過去約180日を取得する。
- 取得済みデータは `data/raw/yahoo/minute/` と `data/raw/yahoo/daily/` にキャッシュする。
- `実データを再取得` ボタンでキャッシュを無視して再取得する。
- Yahooファイナンスの1分足は取得可能期間が短いため、対象日・銘柄によって取得できない場合がある。
- 実データ取得に失敗した場合、サンプルへ黙ってフォールバックせず、日本語エラーを表示して停止する。

### 6.3 サンプルデータ

- `data/raw/minute/` と `data/raw/daily/` にサンプルCSVがなければ自動生成する。
- キオクシアホールディングス `285A` は、2026-06-24付近で約90,000円台になるように生成する。

### 6.4 CSVスキーマ

1分足:

```csv
timestamp,open,high,low,close,volume
2026-06-24 09:00:00,90000,91080,89280,90540,120000
```

日足:

```csv
date,open,high,low,close,volume
2026-06-24,90000,91800,88200,90750,18000000
```

価格単位は円。UI上では `93,730円` のようにカンマ区切りで表示し、`k` 表記は使わない。

## 7. ReplayEngine仕様

- 売買練習中に未来の1分足を見せない。
- `current_state()` は現在時点までの分足だけを返す。
- 対象日より後の日足は読み込まない。
- 当日の日足は、再生済み1分足から動的に作る。
- 前場と後場の間はチャートをリセットせず、x軸上に小さな空白だけを入れる。
- データ末尾に到達したら `done=True` になる。
- `force_close_on_end=True` の場合、最後に未決済建玉を自動決済する。

## 8. 売買・口座仕様

### 8.1 アクション

- 買い
- 空売り
- 決済
- 1分進める
- 再生
- 一時停止
- 最後まで

### 8.2 約定

- 現在足の終値で成行仮想約定する。
- 買い建て中に買いを押すと、買い増しする。
- 空売り建て中に空売りを押すと、売り増しする。
- 同方向の追加建玉は平均建値を再計算して1つの集約ポジションとして扱う。
- 買い建て中に空売りを押した場合、現在の買い建玉を全決済する。
- 空売り建て中に買いを押した場合、現在の空売り建玉を全決済する。
- 決済ボタンは現在の集約ポジションを全決済する。
- 部分決済、ロット別管理、手数料、税金、スリッページ、信用取引規制は現時点では実装しない。

### 8.3 入金額と余力

- 初期入金額はサイドバーで入力する。
- デフォルト入金額は10,000,000円。
- 注文株数はサイドバーで入力する。
- デフォルト注文株数は100株。
- 必要額は `約定価格 * 注文株数` とする。
- 買付余力は `初期入金額 + 確定損益 - 建玉金額` とする。
- 必要額が買付余力を超える場合、日本語エラーを表示して注文を拒否する。

## 10. ネックライン設定仕様

### 10.1 起動時フロー

1. アプリ起動後、まず参照用の日足チャートを必ず表示する。
2. ユーザーは日足チャートを見ながら、サポートライン、レジスタンスライン、直近高値、直近安値、移動平均線、ボリンジャーバンドなどの基準ラインを選ぶ。
3. ラベルと色を選ぶ。
4. クリック設定用チャート上の任意の価格をクリックすると、その価格に水平ネックラインを追加する。
5. 任意本数のラインを追加できる。
6. `最後のラインを削除`、`全ラインを削除` で編集できる。
7. `完了してデイトレードへ` を押すとデイトレード画面へ進む。

### 10.2 ラベル

選択肢:

- サポートライン
- レジスタンスライン
- 直近高値
- 直近安値
- 移動平均線
- ボリンジャーバンド
- その他

`その他` の場合は自由入力ラベルを使う。

### 10.3 表示

- 設定したネックラインは、分足チャート、日足、週足、月足に常時表示する。
- ラインにはラベルと価格を表示する。
- 価格は円表記にする。
- ネックライン設定画面では、通常のPlotly日足チャートを参照用に必ず表示する。
- Streamlit標準のPlotlyチャートでは描画済みshapeを安定してPython側へ戻せないため、ネックライン選定画面は独自のHTML/SVGコンポーネントでクリック価格を取得する。

## 11. チャート仕様

### 11.1 分足チャート

- 1分足データを元に、表示時に3分足、5分足、10分足、30分足、60分足へリサンプルする。
- リサンプル時のOHLCV:
  - open: 最初の始値
  - high: 最大高値
  - low: 最小安値
  - close: 最後の終値
  - volume: 合計出来高
- 分足ごとに移動平均線とボリンジャーバンドを再計算する。
- 約定判定は引き続き1分足単位で行う。
- 表示範囲は現在までの過去10分、30分、60分、または全表示。
- 時間が進んでもローソク足の横幅が極端に狭くならないよう、常に選択範囲だけを表示する。
- 前場と後場の間はチャートをリセットせず、小さな空白で表現する。
- y軸は現在表示されているローソク足、指標、ネックラインを余裕を持って含む範囲に自動調整する。

### 11.2 長期足チャート

- 日足、週足、月足を表示できる。
- 週足は週末基準で集約する。
- 月足は月末基準で集約する。
- 長期足の移動平均線とボリンジャーバンドは選択的に表示できる。
- ネックラインも常時表示する。

### 11.3 トレードマーカー

- 約定したタイミングをチャート上に残す。
- マーカー位置は該当足の実体中央値、つまり `(始値 + 終値) / 2` とする。
- 買い建て、買い決済、空売り建て、空売り決済で色と形を変える。
- 3分足以上を表示している場合は、約定した1分足を含む集約足にマーカーを表示する。

## 12. 指標仕様

### 12.1 出来高加重平均価格

- 分足に表示する。
- 当日ここまでの累積価格出来高を累積出来高で割って算出する。

### 12.2 移動平均線

分足:

- 5本移動平均
- 25本移動平均

長期足:

- 5本移動平均
- 25本移動平均
- 75本移動平均

### 12.3 ボリンジャーバンド

- 中心線は20本移動平均。
- 標準偏差は母標準偏差 `ddof=0`。
- 表示するバンド:
  - +1σ
  - -1σ
  - +2σ
  - -2σ
  - +3σ
  - -3σ

## 14. 自動売買戦略仕様

### 14.1 VWAP + 移動平均ブレイクアウト

- フラット時に、現在値が出来高加重平均価格と5本移動平均を上回り、5本移動平均が25本移動平均を上回る場合に買い。
- 買い建て中に、現在値が出来高加重平均価格または25本移動平均を下回る場合に決済。

### 14.2 ボリンジャー逆張り

- ボリンジャーバンドを使った逆張り戦略として実装する。
- 詳細条件は `src/strategies/bollinger_reversion.py` を正とする。

### 14.3 複合ルール

- 複数指標を組み合わせた戦略として実装する。
- 詳細条件は `src/strategies/combined_rule.py` を正とする。

## 16. GitHub管理方針

- コード、仕様書、README、テスト、環境定義をpushする。
- `data/`、`outputs/`、`__pycache__/`、`.pytest_cache/` などの生成物はpushしない。
- 大きな実データやHTMLレポートは必要になった場合だけ別途管理方針を決める。
- 追加要望を実装した場合は、必ずこの仕様書とREADMEの必要箇所を更新する。

## 17. 既知の制約

- 実売買APIには接続しない。
- Yahooファイナンスの1分足は取得可能期間が短く、過去日付によっては取得できない。
- 手数料、税金、スリッページ、信用取引の厳密な保証金管理は未実装。
- 集約ポジションのみ扱い、ロット別ポジションや部分決済は未実装。
- ネックラインは水平線のみ。
- ネックライン設定はドラッグ描画ではなく、チャートクリックによる価格指定。
- 自動再生はStreamlitのrerunとsleepによる簡易再生。

## 18. ゼロから再作成する手順

1. Python 3.11環境を作成する。
2. `requirements.txt` または `environment.yml` に記載された依存関係を入れる。
3. `src/config.py` にディレクトリ、デフォルト銘柄、デフォルト日付、初期入金額、注文株数を定義する。
4. `src/data/schema.py` に1分足・日足スキーマを定義する。
5. `src/data/sample_data.py` にサンプルデータ生成を実装する。
6. `src/data/providers/yahoo.py` にYahooファイナンス取得、東証ティッカー変換、キャッシュ、メタデータ、日本語エラーを実装する。
7. `src/indicators/` に出来高加重平均価格、移動平均線、ボリンジャーバンドを実装する。
8. `src/simulator/` に注文、ポジション、Broker、ReplayEngine、TradingEnvironmentを実装する。
9. `src/ui/` にStreamlitのサイドバー、操作ボタン、Plotlyチャート、ネックラインクリック用コンポーネントを実装する。
10. `app.py` に起動時ネックライン設定フロー、デイトレード画面、約定バナー、自動再生を実装する。
11. `src/strategies/` にルールベース戦略を実装する。
12. `run_backtest.py` と `src/analysis/` にCLIバックテストとレポート出力を実装する。
13. `tests/` に単体テストを追加する。
14. `README.md` にセットアップ、使い方、注意事項、Codexを使って作成した旨を記載する。
15. ruff、pytest、compileallを通す。
16. Streamlitで起動確認する。
