## 1. プロダクトの目的

日本株の過去の1分足・日足データを使い、スマートフォン上でデイトレードの練習とルールベース戦略の検証を行えるAndroidアプリを作る。

- 通勤中や外出先でも、片手で練習を開始できる。
- 未来の値動きを隠した状態で、買い、空売り、決済の判断を練習できる。
- VWAP、移動平均線、ボリンジャーバンド、重要価格ラインを見ながら判断できる。
- 練習終了後に損益、勝率、取引履歴を振り返れる。
- 初期版は学習・検証専用とし、実際の証券口座や実売買APIには接続しない。

本アプリは投資助言や利益を保証するものではない。表示価格、約定、損益はすべてシミュレーションである。

## 3. 対象ユーザーと利用条件

- 日本株のデイトレードを過去データで練習したい個人ユーザー
- PC版より簡単な操作で、短時間の反復練習をしたいユーザー

## 4. 基本設計方針

Android端末上でPythonやStreamlitを動かさない。AndroidアプリはネイティブUIを担当し、既存Python資産はバックエンドAPIとして再利用する。

```text
Android app
  Jetpack Compose UI
        |
  ViewModel / UseCase
        |
  Repository
    |          |
  Room       HTTPS API
                 |
          Python FastAPI
                 |
  existing data / indicators / simulator / strategies
```

- Android側は表示、操作、画面状態、端末キャッシュを担当する。
- Python APIは株価データ取得、指標計算、リプレイセッション、仮想約定を担当する。
- 売買・損益の正本はサーバー側セッションとし、Android側だけで約定結果を作らない。
- APIが返した結果はRoomへ保存し、閲覧可能な結果はオフラインでも再表示できるようにする。
- 初期版の練習開始には通信を必要とする。開始済みセッションの完全オフライン継続は将来対応とする。

## 7. 画面とナビゲーション

```text
起動
  -> ホーム
       -> 練習設定
            -> 重要価格ライン設定（使用時のみ）
                 -> リプレイ
                      -> 結果
       -> 練習履歴 -> 履歴詳細
       -> 設定
```

### 7.1 ホーム

- `新しい練習を始める` を主ボタンとして表示する。
- 直近の練習結果を最大3件表示する。
- 練習履歴と設定へ移動できる。
- 初回のみ、シミュレーションであり実売買ではない旨を表示する。

### 7.2 練習設定

- 銘柄コード
- 対象日
- データ種別: 実データ / サンプル
- 初期入金額。デフォルト10,000,000円
- 注文株数。デフォルト100株
- 重要価格ラインを使う。デフォルトON
- 表示する指標
- `データを読み込む` ボタン

入力エラーは該当項目の直下に表示する。データ取得中は二重送信を防ぎ、進行中表示とキャンセル操作を提供する。

### 7.3 重要価格ライン設定

- 対象日より前の日足だけを表示する。
- チャートのタップ位置に価格ガイドを表示し、確定操作で水平線を追加する。
- ラベル、色、価格を編集できる。
- サポート、レジスタンス、直近高値、直近安値、移動平均、ボリンジャーバンド、その他を候補とする。
- ピンチズーム、横方向パン、直近60営業日に戻す操作を提供する。
- 追加済みラインの一覧、1件削除、全削除を提供する。
- `練習を開始` でリプレイへ進む。

スマートフォンではタップとパンが競合するため、単純タップだけで即時追加せず、価格選択後の確定操作を挟んで誤追加を防ぐ。

### 7.4 リプレイ

縦画面では上から次の順に配置する。

1. 銘柄、対象日、現在時刻、接続状態
2. 確定損益、含み損益、買付余力
3. チャート
4. 建玉概要
5. 売買操作
6. 時間操作

横画面ではチャートを左、口座・建玉・操作を右へ配置する。画面回転時もセッション、現在時刻、再生状態、選択中の時間足を失わない。

売買操作:

- 買い
- 空売り
- 全決済

時間操作:

- 1分戻る
- 1分進める
- 再生 / 一時停止
- 再生速度: 標準、10倍、30倍、60倍、120倍、240倍
- リセット
- 練習終了

注文は連打防止中、API処理中、セッション終了後に無効化する。約定時はスナックバーだけでなく、売買区分、株数、価格が分かるバナーを表示する。

### 7.5 結果

- 確定損益
- 収益率
- 勝率
- 取引回数
- 最大利益、最大損失
- 開始時刻、終了時刻、練習時間
- 売買マーカー付きチャート
- 取引履歴
- `同じ条件でもう一度`
- `ホームへ戻る`

結果は端末内へ保存する。保存に失敗しても結果画面を失わず、再試行できるようにする。

### 7.6 練習履歴

- 新しい順に、銘柄、日付、損益、勝率、取引回数を表示する。
- 銘柄と期間で絞り込める。
- 履歴詳細から結果画面相当の情報を表示する。
- 履歴を1件削除できる。全削除は確認ダイアログを必須とする。

### 7.7 設定

- テーマ: システム / ライト / ダーク
- デフォルト入金額
- デフォルト注文株数
- デフォルト再生速度
- API接続先。開発ビルドのみ変更可
- アプリ情報、免責事項、オープンソースライセンス

## 8. チャート仕様

- ローソク足は日本株の慣例に合わせ、陽線を赤、陰線を緑、同値をグレーにする。
- 出来高バーも対応するローソク足と同色にする。
- 価格は円、出来高と金額は3桁区切りで表示し、`k` 表記を使わない。
- 1分足から3、5、10、30、60分足へ切り替えられる。
- VWAP、MA5、MA25、MA75、ボリンジャーバンド±1σ、±2σ、±3σを選択表示できる。
- 約定マーカーは買い建て、買い決済、空売り建て、空売り決済を識別できる。
- 重要価格ラインはラベルと価格を表示する。
- ピンチズームと横パンを提供する。リプレイ中の自動追従ON/OFFを切り替えられる。
- 自動追従ONでは現在足の右側に余白を設ける。
- 表示中のローソク足を優先して価格軸を決め、離れた指標や重要価格ラインでローソク足を過度に縮小しない。
- TalkBack利用時は、現在値、始値、高値、安値、出来高、時刻をテキストでも確認できる代替表示を提供する。

## 9. リプレイ・売買仕様

- セッション開始時点は対象日の最初の1分足とする。
- 現在時刻より未来の分足をAPIレスポンスと画面へ露出しない。
- 現在足の終値で成行仮想約定する。
- 同方向の追加注文は平均建値を再計算し、1つの集約ポジションとして扱う。
- 買い建て中の空売り、空売り建て中の買いは現在建玉の全決済として扱う。
- 決済は現在建玉の全数量を対象とする。
- 1分戻る操作では、時刻、建玉、約定履歴、確定損益、完了状態を直前のスナップショットへ戻す。
- 必要額が買付余力を超える注文は拒否し、日本語理由を返す。
- データ末尾で未決済建玉がある場合は、ユーザー確認後に最終価格で決済して終了する。自動再生による末尾到達時は自動決済し、その旨を結果へ記録する。
- 前場と後場は同じチャート上で連続表示する。実時間の昼休み待機はAndroid MVPでは行わない。

## 10. データ仕様

分足:

```json
{
  "timestamp": "2026-06-24T09:00:00+09:00",
  "open": 90000,
  "high": 91080,
  "low": 89280,
  "close": 90540,
  "volume": 120000
}
```

日足:

```json
{
  "date": "2026-06-24",
  "open": 90000,
  "high": 91800,
  "low": 88200,
  "close": 90750,
  "volume": 18000000
}
```

- 価格は円単位の数値として扱う。
- API日時はISO 8601で返し、タイムゾーンを省略しない。
- 日足は対象日の前日までだけを返す。
- Yahooデータ取得に失敗した場合、サンプルへ自動で切り替えない。
- APIはエラーコード、ユーザー向け日本語メッセージ、開発者向け詳細IDを返す。
- 端末にはAPIキー、証券会社認証情報、Yahooの未加工レスポンスを保存しない。

## 11. API仕様 v1

ベースパスは `/api/v1` とする。

| Method | Path | 用途 |
| --- | --- | --- |
| `GET` | `/health` | 接続確認、APIバージョン取得 |
| `GET` | `/symbols/{symbol}` | 銘柄名と利用可能日を取得 |
| `GET` | `/market-data/daily` | 重要価格ライン設定用の日足取得 |
| `POST` | `/replay-sessions` | 練習セッション作成 |
| `GET` | `/replay-sessions/{id}` | 現在状態取得 |
| `POST` | `/replay-sessions/{id}/commands` | 時間操作、注文、決済を実行 |
| `DELETE` | `/replay-sessions/{id}` | 未完了セッション破棄 |
| `GET` | `/replay-sessions/{id}/result` | 完了結果取得 |

`commands` の種別:

```text
STEP_FORWARD
STEP_BACK
RESET
BUY
SELL_SHORT
CLOSE
FINISH
```

各更新コマンドには `command_id` を付ける。同じ `command_id` の再送は二重約定させず、最初と同じ結果を返す。状態更新レスポンスには単調増加する `revision` を含め、古いレスポンスで新しい画面状態を上書きしない。

自動再生はAndroid側が一定間隔で `STEP_FORWARD` を送る。1セッションにつき更新コマンドは直列化し、前の応答前に次の注文・時間操作を送らない。

## 12. ローカル保存

Roomへ次を保存する。

- `PracticeHistoryEntity`: 練習結果の要約
- `TradeEntity`: 約定と決済履歴
- `PracticeChartEntity`: 結果表示に必要な間引き済みチャートデータ
- `KeyPriceLineEntity`: 銘柄ごとの重要価格ライン

DataStoreへテーマ、デフォルト入金額、注文株数、再生速度を保存する。

- アプリ削除時に消えてよいデータだけを保存する。
- スキーマ変更にはRoom Migrationを用意し、安易に破壊的マイグレーションしない。
- 保存済み結果はAPIセッション削除後も閲覧できる。

## 13. 状態・エラー処理

主要画面は `Loading`、`Content`、`Empty`、`Error` の状態を明示する。

- 通信切断: 再試行を提示し、確定済みの画面状態は残す。
- タイムアウト: 同一`command_id`で状態照合または再送し、二重約定を防ぐ。
- APIセッション期限切れ: 練習を終了扱いにせず、復元可否を確認して案内する。
- データなし: 対象日変更を案内し、サンプルへの切り替えを明示選択にする。
- 余力不足、建玉なし、セッション終了済み: 操作理由を日本語で表示する。
- 致命的エラー: 開発者向け詳細IDをコピーできるようにする。スタックトレースや内部URLは通常画面へ出さない。

## 14. セキュリティとプライバシー

- 通信は開発用localhostを除きHTTPSのみとする。
- Android Network Security Configで本番の平文HTTPを禁止する。
- 秘密情報をAPK、BuildConfig、Git管理ファイルへ埋め込まない。
- ログへ注文内容を出す場合も、将来の口座番号やトークンは必ずマスクする。
- 初期版はアカウントを持たず、個人情報を収集しない。
- 分析SDK、クラッシュ収集SDKを導入する場合は、収集内容と無効化方法を仕様へ追記してから実装する。
- リリースビルドは署名し、難読化・縮小設定を有効にする。

## 15. アクセシビリティと操作性

- タップ領域は原則48dp以上とする。
- 文字サイズ拡大時も売買ボタンと損益が欠けない。
- 色だけで売買種別や損益を区別せず、ラベルとアイコンを併用する。
- TalkBackの読み上げ順を、上から下の視覚順と一致させる。
- 買い、空売り、決済は誤操作しにくい間隔を取り、処理中は連打できないようにする。
- 端末の戻る操作で未保存の練習を離れる場合は確認する。

## 19. Git・品質管理方針

- Android開発は `feature/android-app` ブランチで進める。
- 既存ブラウザ版を壊さず、Android/APIの変更は `android/`、`mobile_api/`、関連テストへ分離する。
- 仕様変更、実装、テストは同じ変更単位で同期する。
- Androidの生成物、署名鍵、`local.properties`、秘密情報をGitへ含めない。
- CIではAndroid unit test、lint、assemble、Python API testを実行する。
- mainへ統合する前に、既存Python版のpytestと静的チェックも通す。
